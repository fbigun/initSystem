# fbigun/initSystem 项目规范

## Git 规范

- 提交信息遵循 Conventional Commits 规范（如 `feat:`、`fix:`、`refactor:`、`docs:` 等）
- 单次提交只做一件事，保持提交原子性
- 提交前检查：无调试代码（如 `print`、`debugPrint`）、无敏感信息（如 API Key）、测试通过

## 工作原则

- **编码前思考**：遇到歧义必须先问、先呈现权衡，不许默不作声猜需求
- **简洁优先**：50 行能写完绝不写 200 行，不加未请求的功能，不做一次性抽象
- **精准修改**：只改必须改的地方，不许借"顺手优化"之名碰相邻代码
- **目标驱动执行**：修复 Bug 先写能复现的测试，再让它通过
- **回答简洁准确**：避免冗长解释；复杂概念用类比说明
- **语言一致**：用户使用什么语言提问，就用对应语言回答，默认中文
